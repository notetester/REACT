원래는 React 도 자바처럼 클래스 기반이었다.
그런데 React가 발전하면서 함수형 컴포넌트를 사용하기 시작했음
클래스 컴포넌트에서 사용할 수 있던 것들이 
함수형 컴포넌트에서는 사용 제한 되는 것이 발생했음
이를 해결하기 위한 방법으로  hook 기능를 만들었음
즉, hook은 함수형 컴포넌트에서 React 기능을 "갈고리 처럼 끌어다 쓰는 함수"
이름 useXXXX로 시작, 함수형 컴포넌트 전용, 
hook 반드시 컴포넌트 최상단에서만 호출해야 한다.

1. useState - useState는 React에서 컴포넌트 내부의 상태값을 관리하기 위해 사용하는 Hook이다
   const [상태값, 상태변경함수(set함수)] = useState(초기값)
   상태값을 set함수를 이용해서 변경하면 컴포넌트가 자동으로 리렌더링 한다.

2. useEffect - useEffect는 React 컴포넌트가 렌더링된 이후에 특정 작업을 실행하기 위해 사용하는 Hook이다. 
   useEffect(()=>{
     // 실행할 코드
   },[의존성배열]);
 
   - 의존성배열을 생략하면 레더링 될때마다 실행된다.
   - 의존성배열을 [] 로 만들면 최초 1회만 실행 
   - 의존성배열을 특정값을 넣으면 특정값이 변경될때만 실행
   - 마운트/업데이트/언마운트 처리 

   React 생명주기 
   1) 마운트 : 화면에 처음 나타날 때 
      컴포넌트 생성 -> useState  초기화 -> 랜더링 -> useEffect 실행
   2) 업데이트  : state/pops 변경
      status/props 변경 -> 리랜더링
   3) 언 마운트 : 컨포넌트 제거       

3. useMemo
  - 리랜더링 할 때 계산 결과를 캐싱(기억=메모이제이션) 할 수 있게 해주는 hook
  - 값을 다시 계산하지 말고, 이전 계산해 둔 값을 재사용할 수 있는 hook(불 필요한 재 계산을 막아주는 역할)
    const cachedValue = useMemo(calculateValue, dependencies)
    const cachedValue = useMemo(()=>{
         return 계산할_내용
    },[의존성배열])

   - 계속 바뀌지 않는 계산 결과를 만들 때 (무거운계산(반목문, 정렬, 필터 등)) 
   - 부모에서 props로 전달될 복잡한 값(자식 컴포넌트가 불필요하게 리랜더링 되는 것을 방)

4. useCallback
   - useMemo => 계산된 값을 캐싱
   - useCallback => 함수 자체를 캐싱
   - 자식 컴포넌트에 함수 전달

5. useRef
   - 값을 저장하는 상자 (단, 값이 바뀌어도 리랜더링 안 함)
   1) DOM 직접 접근
   2) 리 렌더링 없이 값 보관
   3) 이전값 기억
   4) 타이머
   
6. useId
   - 고유한 ID 를 자동으로 생성해주는 hook  
     같은 컴포넌트를 여러 번 써도 ID 가 겹치지 않음
   - label의 htmlFor와  input 의 id가 같으면 -> label 클릭 시 input 에 자동 포커스
